# Cowork向け作業指示書

## プロジェクト概要

ファイル構成:

├── index.html

├── css/style.css

├── js/data.js

├── js/app.js

├── README.md

└── .gitignore

## 依頼したい作業

### ステップ1: ローカル動作確認

このフォルダで簡易HTTPサーバーを起動して、index.htmlがブラウザで正しく表示されるか確認してください。

- Pythonが入っていれば `python -m http.server 8000` を実行  
- 入っていなければNode.jsの `npx serve` を試してください  
- 起動後、ブラウザで [http://localhost:8000](http://localhost:8000) を開き、診断が最後まで動くかチェックしてください  
- 動作に問題があれば報告してください

### ステップ2: Gitリポジトリの初期化と初回コミット

このフォルダでGitリポジトリを初期化し、初回コミットしてください。

git init

git add .

git commit \-m "Initial commit: Gem Type 16タイプ宝石診断"

git branch \-M main

Gitのユーザー設定が未設定の場合は、設定するかどうか確認してください。 （個人情報になるので、勝手に設定しないでください）

### ステップ3: GitHubへのプッシュ

以下を私に聞いてから進めてください:

- GitHubのユーザー名  
- 新しく作るリポジトリ名（デフォルト: `16type`）  
- リポジトリを Public / Private のどちらにするか

GitHub CLIの `gh` コマンドが使えれば、それでリポジトリ作成 → push まで一気に進められます。 使えなければ、私にGitHubで空リポジトリを作るよう案内してから、`git remote add origin ...` → `git push -u origin main` を実行してください。

### ステップ4: デプロイ（任意・選択肢を提示してから）

以下のどれが良いか私に確認してください:

1. **GitHub Pages**: 追加作業ほぼゼロ。リポジトリのSettings → Pagesから設定するだけ  
2. **Vercel**: `vercel` コマンドで一発デプロイ可能（要アカウント）  
3. **Netlify**: `netlify deploy` で可能（要アカウント）  
4. デプロイしない

選ばれた方法に従って実行してください。アカウントログインが必要な場合は、その都度確認してください。
